Architecture

Three containers, one network, zero tolerance for misconfiguration. Each service lives in its own hardened environment.

Reverse Proxy
NGINX
Entry point of the stack. Handles TLSv1.2/1.3 termination, proxies traffic to WordPress, serves static assets. No other path into the cluster.
:443 — HTTPS only
CMS
WordPress + PHP-FPM
Content layer. PHP-FPM processes requests forwarded by NGINX via FastCGI. Admin and one user pre-seeded. Connected to MariaDB on internal network.
:9000 — FastCGI (internal)
Database
MariaDB
Persistent relational store. Two users — root with env-sourced credentials, and an application user for WordPress. Data survives container restarts via named volume.
:3306 — MySQL (internal)

Quick Start

Deploy

01
Clone the repository
Get the project source. Ensure Docker Engine & Docker Compose are installed.
git clone https://github.com/<login>/inception.git && cd inception
02
Configure environment
Copy the example env file and fill in credentials. Never commit secrets to Git.
cp .env.example .env && $EDITOR .env
03
Build & launch
Compose will build all three images from their Dockerfiles and start the stack.
make
04
Map the domain
Add your login domain to /etc/hosts so the browser resolves it locally.
echo "127.0.0.1 <login>.42.fr" | sudo tee -a /etc/hosts
05
Open in browser
Navigate to your domain over HTTPS. Accept the self-signed cert warning.
https://<login>.42.fr
